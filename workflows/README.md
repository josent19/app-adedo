# Workflows

Markdown SOPs that define what to do and how. These are the instructions I follow.

## Structure of a workflow

Each workflow file should include:

- **Objective**: What this workflow accomplishes
- **Inputs**: What is needed to start
- **Steps**: Ordered list of actions (tools to call, decisions to make)
- **Outputs**: What is produced and where it goes
- **Edge cases**: Known failures and how to handle them
- **Notes**: Lessons learned, rate limits, quirks discovered over time

## Naming convention

`workflows/verb_noun.md` — e.g., `scrape_website.md`, `generate_report.md`, `send_email.md`
